# Thor Stream Butler v0.4.0-alpha.1 — Hosts, profiles, and local backups / Hosts, Profile und lokale Sicherungen

## English

### Highlights

- Link a dashboard tile to a local streaming host. Its pre-launch check now
  includes host reachability and can send Wake-on-LAN from the result dialog.
- Discover compatible Sunshine/Moonlight hosts on demand through Android NSD.
- Store a resolution, frame-rate, and bitrate profile per tile and receive a
  conservative recommendation based on the full network assessment.
- Export or import the local configuration as versioned JSON through Android's
  document picker. Measurement history remains optional.
- Filter the history, compare matching connection contexts, and view a compact
  latency trend.
- Use a hardened local-first build with transactional configuration import,
  IPv6 host support, complete privacy notices, and an adaptive launcher icon.
- Follow prepared bilingual physical-device and store/Data-safety release gates.

### Changes

- Adds a privacy-safe diagnostic log containing only timestamps and predefined
  event types, limited to 200 entries and disabled by default.
- Improves controller focus restoration, keeps the tile focus frame visible for
  nested controls, supports D-pad tile ordering, and confirms tile removal.
- Migrates Room schema 1 to 2 to store host assignments and streaming profiles.
- Updates the stable build stack to Gradle 9.6.1, Kotlin 2.4.10, Compose BOM
  2026.06.01, AndroidX Core 1.19.0, Hilt Navigation 1.4.0, and Coroutines 1.11.0.
- Adds a local production-signing setup helper while keeping stable releases
  fail-closed when the four GitHub signing secrets are incomplete.
- Evaluates real DNS, Wi-Fi link-rate, and optional download measurements in
  addition to latency, jitter, packet loss, signal, transport, and host reachability.
- Limits imported files and collection sizes, validates cross-references and
  numeric ranges, honors cancellation, and rolls Room data back if DataStore
  cannot complete the import or the operation is cancelled.
- Caps history at 100 measurements and safely handles unexpected network,
  storage, discovery, and ViewModel failures without swallowing cancellation.
- Normalizes stored diagnostic targets and limits, and safely maps unknown future
  Room enum values instead of crashing while local data is loaded.
- Improves portrait navigation, system-bar contrast, controller focus in the
  app picker, and confirmation before deleting hosts or history.
- Keeps partial diagnostic failures gray so they cannot auto-launch an app and
  reports IPv6 local addresses or gateways when IPv4 is unavailable.
- Keeps release passwords out of `keytool` command arguments and verifies each
  individual GitHub signing-secret upload.
- Enables R8 and resource shrinking for releases; CI produces APK and AAB,
  runs instrumentation tests on Android 9/API 28 and Android 15/API 35, pins
  workflow actions to reviewed commits, rejects new moderate-or-higher
  dependency vulnerabilities, and
  rejects an unsigned bundle before publication.
- Verifies the Gradle wrapper and resolved artifacts with reviewed SHA-256
  metadata for Windows, Linux, and macOS build hosts.

### Fixes

- History trends no longer compare unrelated Wi-Fi networks, transports, or
  host tests.
- Linked-host checks no longer depend on the dashboard screen actively
  collecting UI state.
- Deleted hosts are detached from their dashboard tiles instead of leaving a
  stale assignment.
- DNS failures can no longer abort a diagnostic run, unresolved NSD services
  time out, and missing permissions remain a non-crashing unavailable value.
- Room migration schemas are packaged into device tests, and the serialization
  runtime is aligned with Room's migration reader.
- Invalid persisted diagnostic limits and forward-version database enums no
  longer abort settings, dashboard, host, or history loading.

### Migration

- Existing Room schema 1 data is migrated automatically to schema 2.
- Configuration import replaces tiles, hosts, and settings. History is replaced
  only when included; an invalid or incomplete import leaves existing data intact.
- No manual migration is needed when the signing certificate matches the installed build.

### Known limitations

- **Signature:** If the repository still has no production-signing secrets,
  this prerelease is published with an explicitly named Android debug
  signature. It may not update an older debug alpha. Uninstalling an older
  alpha removes its settings, hosts, and history.
- **Hardware validation:** 40 JVM tests and 14 Android instrumentation tests,
  D-pad/touch flows, permission denial, optimized release startup, and a real
  target-app launch passed locally on an Android 15 emulator. The device suite
  also passes in CI on Android 9/API 28 and Android 15/API 35. Physical handheld,
  controller, vendor Wi-Fi/Ethernet, Wake-on-LAN, and long-session tests remain pending.
- **Discovery:** Only hosts advertising `_nvstream._tcp` on the same local
  network are found. Routers or Android devices may filter discovery traffic.
- **Profiles:** Recommendations are guidance. Third-party apps are not modified
  unless they expose a supported public configuration API.
- **Backup privacy:** Export files can contain host addresses, MAC addresses,
  and, when selected, SSIDs and measurements. Keep them private.

### Installation or update

1. Download the APK and matching `.sha256` file attached to this release. The
   AAB is intended for managed/store deployment and is not directly sideloaded.
2. Verify the SHA-256 checksum before installation.
3. If Android rejects the update because the signing certificate differs,
   export the old configuration if supported, uninstall the old alpha, and
   install this APK. Uninstalling removes all local app data.
4. Android migrates an existing compatible Room database automatically.

### Privacy note

Thor Stream Butler contains no advertising, tracking, analytics, or telemetry.
Host discovery runs only after user action. Configuration files are read and
written only at the document location selected by the user. Diagnostic logs
cannot contain IP addresses, hostnames, SSIDs, or MAC addresses.
The optional download test contacts Cloudflare over HTTPS and exposes the
connection's public IP to that provider while it runs. See the full
[privacy policy](../privacy/index.html).

## Deutsch

### Highlights

- Verknüpfe eine Dashboard-Kachel mit einem lokalen Streaming-Host. Der
  Netzwerkcheck vor dem Start prüft nun dessen Erreichbarkeit und kann im
  Ergebnisdialog Wake-on-LAN senden.
- Kompatible Sunshine-/Moonlight-Hosts werden auf Wunsch über Android NSD gesucht.
- Pro Kachel lassen sich Auflösung, Bildrate und Bitrate speichern; eine
  konservative Empfehlung basiert auf der vollständigen Netzwerkbewertung.
- Die lokale Konfiguration kann als versioniertes JSON über die
  Android-Dateiauswahl exportiert oder importiert werden. Die Messhistorie
  bleibt optional.
- Filtere die Historie, vergleiche passende Verbindungskontexte und sieh einen
  kompakten Latenzverlauf.
- Nutze einen gehärteten Local-first-Build mit transaktionalem
  Konfigurationsimport, IPv6-Hosts, vollständigen Datenschutzhinweisen und
  adaptivem Launcher-Icon.
- Nutze vorbereitete zweisprachige Hardware- und Store-/Data-Safety-Release-Gates.

### Änderungen

- Ergänzt ein datenschutzsicheres Diagnoseprotokoll nur aus Zeitpunkten und
  fest definierten Ereignistypen, begrenzt auf 200 Einträge und standardmäßig aus.
- Verbessert Fokuswiederherstellung und verschachtelte Fokusrahmen, ermöglicht
  D-Pad-Kachelsortierung und bestätigt das Entfernen einer Kachel.
- Migriert Room-Schema 1 auf 2 für Host-Zuordnungen und Streaming-Profile.
- Aktualisiert den stabilen Build-Stack auf Gradle 9.6.1, Kotlin 2.4.10,
  Compose BOM 2026.06.01, AndroidX Core 1.19.0, Hilt Navigation 1.4.0 und
  Coroutines 1.11.0.
- Ergänzt einen lokalen Helfer zur Produktionssignierung; stabile Releases
  bleiben bei unvollständigen GitHub-Signing-Secrets sicher blockiert.
- Bewertet echte DNS-, WLAN-Linkraten- und optionale Downloadmessungen zusätzlich
  zu Latenz, Jitter, Paketverlust, Signal, Transport und Host-Erreichbarkeit.
- Begrenzt Importdatei und Sammlungen, validiert Querverweise sowie Zahlenbereiche
  und respektiert Cancellation. Room-Daten werden wiederhergestellt, wenn DataStore
  den Import nicht abschließen kann oder der Vorgang abgebrochen wird.
- Begrenzt die Historie auf 100 Messungen und behandelt unerwartete Netzwerk-,
  Speicher-, Such- und ViewModel-Fehler sicher, ohne Cancellation zu verschlucken.
- Normalisiert gespeicherte Diagnoseziele und Grenzwerte; unbekannte zukünftige
  Room-Enum-Werte werden beim Laden lokaler Daten absturzsicher abgebildet.
- Verbessert Portrait-Navigation, Systemleisten-Kontrast, Controller-Fokus in
  der App-Auswahl und Bestätigungen vor dem Löschen von Hosts oder Historie.
- Hält teilweise fehlgeschlagene Diagnosen grau, sodass sie keine App automatisch
  starten, und zeigt IPv6-Adressen oder -Gateways an, wenn IPv4 fehlt.
- Hält Release-Passwörter aus `keytool`-Kommandoargumenten heraus und prüft
  jeden einzelnen Upload eines GitHub-Signing-Secrets.
- Aktiviert R8 und Ressourcenverkleinerung; CI erzeugt APK und AAB, führt
  Instrumentationstests auf Android 9/API 28 und Android 15/API 35 aus und setzt
  Workflow-Aktionen auf geprüfte Commits fest, blockiert neue
  Abhängigkeitsschwachstellen ab mittlerer Schwere
  und weist ein unsigniertes Bundle vor der Veröffentlichung zurück.
- Prüft Gradle Wrapper und aufgelöste Artefakte per kontrollierter
  SHA-256-Metadaten für Windows-, Linux- und macOS-Buildhosts.

### Fehlerbehebungen

- Historientrends vergleichen keine unterschiedlichen WLAN-Netze,
  Transportarten oder Host-Tests mehr miteinander.
- Die Prüfung eines verknüpften Hosts hängt nicht mehr davon ab, dass der
  Dashboard-Screen gerade UI-Zustand sammelt.
- Gelöschte Hosts werden von Dashboard-Kacheln gelöst, statt eine veraltete
  Zuordnung zu hinterlassen.
- DNS-Fehler brechen einen Diagnoselauf nicht mehr ab, nicht aufgelöste
  NSD-Dienste laufen in ein Timeout und fehlende Berechtigungen bleiben ein
  absturzsicherer nicht verfügbarer Wert.
- Room-Migrationsschemata werden in Gerätetests eingebunden; die
  Serialisierungslaufzeit ist an Rooms Migrationsleser angeglichen.
- Ungültige gespeicherte Diagnosegrenzen und Datenbank-Enums aus zukünftigen
  Versionen brechen Einstellungen, Dashboard, Hosts oder Historie nicht mehr ab.

### Migration

- Vorhandene Daten aus Room-Schema 1 werden automatisch auf Schema 2 migriert.
- Ein Konfigurationsimport ersetzt Kacheln, Hosts und Einstellungen. Die Historie
  wird nur bei enthaltenen Daten ersetzt; ein ungültiger oder unvollständiger
  Import lässt vorhandene Daten unverändert.
- Bei identischem Signaturzertifikat ist keine manuelle Migration erforderlich.

### Bekannte Einschränkungen

- **Signatur:** Sind noch keine produktiven Signing-Secrets eingerichtet, wird
  dieses Prerelease mit einer eindeutig benannten Android-Debug-Signatur
  veröffentlicht. Es kann eine ältere Debug-Alpha eventuell nicht aktualisieren.
  Die Deinstallation einer alten Alpha löscht Einstellungen, Hosts und Historie.
- **Hardwareprüfung:** 40 JVM-Tests und 14 Android-Instrumentationstests,
  D-Pad-/Touch-Abläufe, Berechtigungsverweigerung, optimierter Release-Start und
  ein echter Ziel-App-Start bestanden lokal auf einem Android-15-Emulator. Die
  Gerätesuite besteht in CI außerdem auf Android 9/API 28 und Android 15/API 35.
  Tests auf physischem Handheld und Controller sowie Hersteller-WLAN/Ethernet,
  Wake-on-LAN und lange Sitzungen stehen noch aus.
- **Erkennung:** Gefunden werden nur Hosts, die `_nvstream._tcp` im selben
  lokalen Netz ankündigen. Router oder Android-Geräte können Discovery-Verkehr filtern.
- **Profile:** Empfehlungen dienen als Orientierung. Drittanbieter-Apps werden
  ohne unterstützte öffentliche Konfigurations-API nicht verändert.
- **Datenschutz der Sicherung:** Exportdateien können Host-Adressen,
  MAC-Adressen und bei Auswahl auch SSIDs und Messwerte enthalten. Bewahre sie privat auf.

### Installation oder Update

1. Lade die APK und die zugehörige `.sha256`-Datei aus diesem Release herunter.
   Das AAB ist für verwaltete/Store-Bereitstellung gedacht und nicht direkt installierbar.
2. Prüfe vor der Installation die SHA-256-Prüfsumme.
3. Lehnt Android das Update wegen einer anderen Signatur ab, exportiere – falls
   unterstützt – die alte Konfiguration, deinstalliere die alte Alpha und
   installiere diese APK. Eine Deinstallation löscht alle lokalen App-Daten.
4. Android migriert eine vorhandene kompatible Room-Datenbank automatisch.

### Datenschutzhinweis

Thor Stream Butler enthält keine Werbung, kein Tracking, keine Analytics und
keine Telemetrie. Die Host-Suche startet nur nach einer Benutzeraktion.
Konfigurationsdateien werden ausschließlich am vom Benutzer ausgewählten Ort
gelesen und geschrieben. Diagnoseprotokolle können keine IP-Adressen,
Hostnamen, SSIDs oder MAC-Adressen enthalten.
Der optionale Downloadtest kontaktiert Cloudflare per HTTPS; dabei sieht dieser
Anbieter während des Tests technisch die öffentliche IP der Verbindung. Siehe
die vollständige [Datenschutzerklärung](../de/datenschutz/index.html).

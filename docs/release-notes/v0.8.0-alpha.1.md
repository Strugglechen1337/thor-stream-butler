# Thor Stream Butler v0.8.0-alpha.1 — Host port check / Host-Portprüfung

## English

### Highlights

- **Port check for your hosts:** every saved host now has a **Check ports**
  action. It probes a curated list of common local streaming ports —
  Sunshine/GameStream (47984, 47989, 47990, 48010), PlayStation Remote Play
  (9295), and Steam Remote Play (27036) — plus the host's own configured port,
  and reports each as open or closed. Handy when a streaming host is reachable
  but a session still will not start.

### Changes

- Only the single host you entered is probed. There is no network, subnet, or
  address-range scanning; a closed or filtered port is a result, not an error.
- TCP probes run with bounded concurrency so the check stays quick without
  flooding the host.

### Known limitations

- **Update path:** This build is signed with the permanent production key and
  updates v0.4.0-alpha.1 or newer in place. Debug-signed previews (v0.3 and
  earlier) must be uninstalled first; export the configuration beforehand if
  needed.
- **Hardware validation:** installation on physical handheld hardware was not
  available during this release run.
- An open port only means the port accepts a TCP connection; it does not
  confirm that the streaming service behind it is fully configured.
- Automatic background monitoring and notifications for unstable connections
  are not implemented yet.

### Installation

1. Update directly over v0.4.0-alpha.1 or newer, or uninstall any older
   debug-signed alpha first (export the configuration beforehand if needed).
2. Download `ThorStreamButler-v0.8.0-alpha.1.apk` and its `.sha256` checksum
   from the release assets.
3. Verify the checksum, allow installation from your file manager or browser,
   and install the APK on Android 9 or newer.

## Deutsch

### Highlights

- **Portprüfung für deine Hosts:** Jeder gespeicherte Host hat jetzt eine
  Aktion **Ports prüfen**. Sie prüft eine kuratierte Liste gängiger lokaler
  Streaming-Ports — Sunshine/GameStream (47984, 47989, 47990, 48010),
  PlayStation Remote Play (9295) und Steam Remote Play (27036) — sowie den
  konfigurierten Port des Hosts und meldet je Port „offen“ oder „geschlossen“.
  Praktisch, wenn ein Streaming-Host zwar erreichbar ist, eine Sitzung aber
  trotzdem nicht startet.

### Änderungen

- Es wird nur der eine eingetragene Host geprüft. Es gibt keinen Netzwerk-,
  Subnetz- oder Adressbereich-Scan; ein geschlossener oder gefilterter Port ist
  ein Ergebnis, kein Fehler.
- Die TCP-Prüfungen laufen mit begrenzter Parallelität, damit die Prüfung
  schnell bleibt, ohne den Host zu überlasten.

### Bekannte Einschränkungen

- **Update-Pfad:** Dieser Build ist mit dem dauerhaften Produktionsschlüssel
  signiert und aktualisiert v0.4.0-alpha.1 oder neuer direkt. Debug-signierte
  Vorschauen (v0.3 und früher) müssen zuerst deinstalliert werden; exportiere
  bei Bedarf vorher die Konfiguration.
- **Hardware-Prüfung:** Eine Installation auf physischer Handheld-Hardware
  stand für diesen Release-Lauf nicht zur Verfügung.
- Ein offener Port bedeutet nur, dass der Port eine TCP-Verbindung annimmt; er
  bestätigt nicht, dass der dahinterliegende Streaming-Dienst vollständig
  eingerichtet ist.
- Automatische Hintergrundüberwachung und Benachrichtigungen bei instabiler
  Verbindung sind noch nicht umgesetzt.

### Installation

1. Aktualisiere direkt über v0.4.0-alpha.1 oder neuer oder deinstalliere
   zuerst eine ältere debug-signierte Alpha (bei Bedarf vorher die
   Konfiguration exportieren).
2. Lade `ThorStreamButler-v0.8.0-alpha.1.apk` und die zugehörige
   `.sha256`-Prüfsumme aus den Release-Dateien herunter.
3. Prüfe die Prüfsumme, erlaube die Installation aus Dateimanager oder
   Browser und installiere die APK auf Android 9 oder neuer.

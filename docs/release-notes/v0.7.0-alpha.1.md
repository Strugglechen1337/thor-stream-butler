# Thor Stream Butler v0.7.0-alpha.1 — Wired and wireless profiles / Kabel- und Funkprofile

## English

### Highlights

- **Separate Wi-Fi and Ethernet profiles:** every dashboard tile can now store
  a dedicated Ethernet profile (resolution, FPS, bitrate) in addition to its
  default profile. Wired connections usually carry more bandwidth — set a
  higher-quality profile for the dock and keep a conservative one for Wi-Fi.
- The pre-launch network check applies the Ethernet profile automatically on
  wired connections and shows which profile is active. All other transports
  keep using the default profile.
- Tiles indicate a configured LAN profile, and the JSON configuration
  export/import carries the optional Ethernet profile along.

### Changes

- Room schema 3 adds the optional Ethernet profile columns with a tested
  migration from schema 2; existing tiles keep their stored profile unchanged.
- The documented JVM test count in the README was corrected to 47 in both
  languages.

### Known limitations

- **Update path:** This build is signed with the permanent production key and
  updates v0.4.0-alpha.1 or newer in place. Debug-signed previews (v0.3 and
  earlier) must be uninstalled first; export the configuration beforehand if
  needed.
- **Hardware validation:** installation on physical handheld hardware was not
  available during this release run.
- The Ethernet profile is applied by the pre-launch check; third-party
  streaming apps without a public API cannot be reconfigured automatically.
- Automatic background monitoring and notifications for unstable connections
  are not implemented yet.

### Installation

1. Update directly over v0.4.0-alpha.1 or newer, or uninstall any older
   debug-signed alpha first (export the configuration beforehand if needed).
2. Download `ThorStreamButler-v0.7.0-alpha.1.apk` and its `.sha256` checksum
   from the release assets.
3. Verify the checksum, allow installation from your file manager or browser,
   and install the APK on Android 9 or newer.

## Deutsch

### Highlights

- **Getrennte WLAN- und Ethernet-Profile:** Jede Dashboard-Kachel kann jetzt
  zusätzlich zum Standardprofil ein eigenes Ethernet-Profil (Auflösung, FPS,
  Bitrate) speichern. Kabelverbindungen bieten meist mehr Bandbreite — hinterlege
  ein hochwertigeres Profil für das Dock und ein konservatives für WLAN.
- Der Netzwerkcheck vor dem Start wendet das Ethernet-Profil bei
  Kabelverbindung automatisch an und zeigt das aktive Profil. Alle anderen
  Transporte nutzen weiterhin das Standardprofil.
- Kacheln kennzeichnen ein gesetztes LAN-Profil, und der
  JSON-Konfigurations-Export/-Import nimmt das optionale Ethernet-Profil mit.

### Änderungen

- Room-Schema 3 ergänzt die optionalen Ethernet-Profilspalten mit getesteter
  Migration von Schema 2; bestehende Kacheln behalten ihr gespeichertes Profil
  unverändert.
- Die dokumentierte JVM-Testanzahl in der README wurde in beiden Sprachen auf
  47 korrigiert.

### Bekannte Einschränkungen

- **Update-Pfad:** Dieser Build ist mit dem dauerhaften Produktionsschlüssel
  signiert und aktualisiert v0.4.0-alpha.1 oder neuer direkt. Debug-signierte
  Vorschauen (v0.3 und früher) müssen zuerst deinstalliert werden; exportiere
  bei Bedarf vorher die Konfiguration.
- **Hardware-Prüfung:** Eine Installation auf physischer Handheld-Hardware
  stand für diesen Release-Lauf nicht zur Verfügung.
- Das Ethernet-Profil wird vom Netzwerkcheck angewendet; Streaming-Apps von
  Drittanbietern ohne öffentliche API können nicht automatisch umkonfiguriert
  werden.
- Automatische Hintergrundüberwachung und Benachrichtigungen bei instabiler
  Verbindung sind noch nicht umgesetzt.

### Installation

1. Aktualisiere direkt über v0.4.0-alpha.1 oder neuer oder deinstalliere
   zuerst eine ältere debug-signierte Alpha (bei Bedarf vorher die
   Konfiguration exportieren).
2. Lade `ThorStreamButler-v0.7.0-alpha.1.apk` und die zugehörige
   `.sha256`-Prüfsumme aus den Release-Dateien herunter.
3. Prüfe die Prüfsumme, erlaube die Installation aus Dateimanager oder
   Browser und installiere die APK auf Android 9 oder neuer.

# Thor Stream Butler v0.2.0-alpha.1 — Bilingual Preview / Zweisprachige Vorschau

## English

### Highlights

- The complete in-app interface is now available in **English and German**.
  English is the new default language; German follows the Android system
  language or the per-app language setting (Android 13+).
- All screens, dialogs, error messages, diagnostic steps, and network quality
  assessments are localized.
- Release signing infrastructure is in place for upcoming production-signed
  builds.
- Still no account, cloud backend, advertising, tracking, or telemetry.

### Changes

- Moves every hardcoded UI string into English and German string resources and
  registers both languages for the Android per-app language setting.
- Formats measured values (latency, jitter, packet loss, download) with the
  active locale instead of always using German decimal formatting.
- Uses locale-independent identifiers for diagnostic progress steps
  internally, keeping stored data independent of the display language.
- Adds Gradle release signing (local `signing/keystore.properties` or CI
  secrets) and a tag-triggered Release workflow that builds a signed APK with
  a SHA-256 checksum once signing secrets are configured.

### Fixes

- A failed pre-launch network check no longer stores an invented "no
  connection" measurement in the history; the error is shown directly in the
  launch dialog instead.
- Installed-app lookups for dashboard tiles no longer run on the main thread.
- The network quality dot on dashboard tiles is drawn as a clean filled circle.

### Known limitations

- **Development signature:** This APK is still signed with the standard
  Android debug certificate. It is a prerelease for testing, not a
  production-signed stable build.
- **Update path:** This APK cannot update v0.1.0-alpha.1 in place. Uninstall
  the previous alpha first; Android will then remove its local settings,
  configured hosts, and measurement history.
- **Hardware validation:** The APK build, package metadata, and signature were
  verified. No physical Android handheld was connected for installation during
  this release run.
- Automatic Sunshine and Moonlight host discovery is not implemented yet.
- Website screenshots are sanitized interface previews, not real-device
  captures.
- Some Wi-Fi and network values may be unavailable because of Android version,
  device support, permissions, or platform privacy restrictions. The app marks
  such values as not measurable.

### Installation

1. Uninstall any previously installed Thor Stream Butler alpha.
2. Download `ThorStreamButler-v0.2.0-alpha.1-debug.apk` and its `.sha256`
   checksum from the release assets.
3. Verify the checksum, allow installation from your file manager or browser,
   and install the APK on Android 9 or newer.

## Deutsch

### Highlights

- Die komplette App-Oberfläche ist jetzt auf **Englisch und Deutsch**
  verfügbar. Englisch ist die neue Standardsprache; Deutsch folgt der
  Android-Systemsprache bzw. der App-Sprach-Einstellung (Android 13+).
- Alle Screens, Dialoge, Fehlermeldungen, Diagnoseschritte und
  Netzwerk-Qualitätsbewertungen sind lokalisiert.
- Die Release-Signing-Infrastruktur für kommende produktiv signierte Builds
  ist vorbereitet.
- Weiterhin kein Konto, kein Cloud-Backend, keine Werbung, kein Tracking,
  keine Telemetrie.

### Änderungen

- Verschiebt alle hartkodierten UI-Texte in englische und deutsche
  String-Ressourcen und registriert beide Sprachen für die
  App-Sprach-Einstellung von Android.
- Formatiert Messwerte (Latenz, Jitter, Paketverlust, Download) mit der
  aktiven Sprache statt immer mit deutscher Dezimalformatierung.
- Verwendet intern sprachunabhängige Kennungen für
  Diagnose-Fortschrittsschritte; gespeicherte Daten bleiben damit von der
  Anzeigesprache unabhängig.
- Ergänzt Gradle-Release-Signing (lokal `signing/keystore.properties` oder
  CI-Secrets) und einen tag-gesteuerten Release-Workflow, der nach
  Einrichtung der Signing-Secrets eine signierte APK mit SHA-256-Prüfsumme
  baut.

### Fehlerbehebungen

- Ein fehlgeschlagener Netzwerkcheck vor dem App-Start speichert keine
  erfundene „keine Verbindung“-Messung mehr in der Historie; der Fehler wird
  stattdessen direkt im Start-Dialog angezeigt.
- Die Abfrage installierter Apps für Dashboard-Kacheln läuft nicht mehr auf
  dem Main-Thread.
- Der Netzwerkqualitäts-Punkt auf den Dashboard-Kacheln wird als sauber
  gefüllter Kreis gezeichnet.

### Bekannte Einschränkungen

- **Entwicklungs-Signatur:** Diese APK ist weiterhin mit dem
  Standard-Debug-Zertifikat von Android signiert. Sie ist ein Prerelease zum
  Testen, kein produktiv signiertes stabiles Release.
- **Update-Pfad:** Diese APK kann v0.1.0-alpha.1 nicht direkt aktualisieren.
  Deinstalliere die vorherige Alpha zuerst; Android entfernt dabei deren
  lokale Einstellungen, konfigurierte Hosts und die Messhistorie.
- **Hardware-Prüfung:** APK-Build, Paket-Metadaten und Signatur wurden
  geprüft. Für diesen Release-Lauf war kein physisches Android-Handheld zur
  Installation angeschlossen.
- Automatische Sunshine- und Moonlight-Host-Erkennung ist noch nicht
  umgesetzt.
- Website-Screenshots sind bereinigte Oberflächenvorschauen, keine Aufnahmen
  von echten Geräten.
- Einzelne WLAN- und Netzwerkwerte können je nach Android-Version,
  Geräteunterstützung, Berechtigungen oder Datenschutz-Einschränkungen nicht
  verfügbar sein. Die App kennzeichnet solche Werte als nicht messbar.

### Installation

1. Deinstalliere eine zuvor installierte Thor-Stream-Butler-Alpha.
2. Lade `ThorStreamButler-v0.2.0-alpha.1-debug.apk` und die zugehörige
   `.sha256`-Prüfsumme aus den Release-Dateien herunter.
3. Prüfe die Prüfsumme, erlaube die Installation aus Dateimanager oder
   Browser und installiere die APK auf Android 9 oder neuer.

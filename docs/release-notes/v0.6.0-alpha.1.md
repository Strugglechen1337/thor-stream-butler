# Thor Stream Butler v0.6.0-alpha.1 — Streaming session timer / Streaming-Session-Timer

## English

### Highlights

- **Local session timer:** launching a streaming app from the dashboard records
  a local start marker. Returning to Thor Stream Butler completes the session
  and shows the latest app, start time, and duration directly on the dashboard.
- **Permissionless by design:** the timer uses no usage-access permission,
  background tracking, account, telemetry, or cloud service.
- **Resilient local state:** the active marker and latest completed session use
  a dedicated DataStore and survive Android reclaiming the app process.

### Changes

- Sessions shorter than one minute are treated as accidental launches and are
  not recorded.
- Sessions longer than twelve hours are treated as stale markers caused by a
  reboot, crash, or clock change and are discarded safely.
- Session persistence is best effort: a storage error never blocks launching a
  streaming app or opening the dashboard.

### Fixes

- None.

### Migration

- No manual migration is required. The dedicated local session DataStore is
  created automatically on first use.
- v0.6.0-alpha.1 uses the same permanent production signing key and updates
  v0.5.0-alpha.1 in place.

### Known limitations

- **Hardware validation:** installation and controller/network testing on a
  physical AYN/Odin/Retroid-class handheld remain pending.
- **Approximate duration:** Android exposes no precise third-party app usage
  duration without additional sensitive permission. The timer measures from a
  successful launch until the dashboard becomes visible again, so interruptions
  outside the streaming app can be included.
- Only the latest completed session is retained; this release does not add a
  full session-history view.
- Debug-signed previews (v0.3 and earlier) cannot be updated in place. Export
  the configuration if needed before uninstalling such a preview.

### Validation

- 47 JVM tests passed with 0 failures and 0 skipped tests.
- 14 Android instrumentation/Compose tests passed on Android 9/API 28 and the
  same 14 passed on Android 15/API 35, with 0 failures or skipped tests.
- Debug and release lint report 0 issues; debug APK, optimized release APK,
  release AAB, and Android test APK build successfully.
- APK and AAB use the permanent production update certificate. Certificate
  SHA-256: `04:4D:A3:96:49:79:92:A7:95:FA:2A:B1:4D:C3:6D:83:95:E6:2E:8A:E6:E4:AC:DB:4D:EC:FE:78:2D:DA:B3:8F`.
- Physical handheld validation remains pending as documented above.

### Installation or update

1. Update directly over production-signed v0.4.0-alpha.1 or newer. For a
   debug-signed v0.3 or earlier preview, export the configuration if needed,
   uninstall it, and then install this release.
2. Download `ThorStreamButler-v0.6.0-alpha.1.apk` and its `.sha256` file. The
   AAB is for managed/store deployment, not direct sideloading.
3. Verify the published SHA-256 checksum and install the APK on Android 9 or newer.

### Privacy note

Thor Stream Butler contains no advertising, tracking, or telemetry. Session
markers, network measurements, configured hosts, settings, and history stay
local on the device unless the user explicitly exports configuration data. The
optional HTTPS download test contacts the documented provider. Full policy:
<https://strugglechen1337.github.io/thor-stream-butler/privacy/>.

## Deutsch

### Highlights

- **Lokaler Session-Timer:** Beim Start einer Streaming-App aus dem Dashboard
  wird lokal eine Startmarke gespeichert. Die Rückkehr zu Thor Stream Butler
  schließt die Session ab und zeigt die letzte App, Startzeit und Dauer direkt
  im Dashboard.
- **Bewusst berechtigungsfrei:** Der Timer verwendet weder Zugriff auf
  Nutzungsdaten noch Hintergrund-Tracking, Konto, Telemetrie oder Cloud-Dienst.
- **Robuster lokaler Zustand:** Aktive Marke und letzte abgeschlossene Session
  nutzen einen eigenen DataStore und überleben das Beenden des App-Prozesses
  durch Android.

### Änderungen

- Sessions unter einer Minute gelten als versehentliche Starts und werden
  nicht gespeichert.
- Sessions über zwölf Stunden gelten als veraltete Marken durch Neustart,
  Absturz oder Uhrzeitänderung und werden sicher verworfen.
- Die Session-Speicherung arbeitet nach Best Effort: Ein Speicherfehler blockiert
  weder den Start einer Streaming-App noch das Öffnen des Dashboards.

### Fehlerbehebungen

- Keine.

### Migration

- Es ist keine manuelle Migration erforderlich. Der eigene lokale
  Session-DataStore wird bei der ersten Verwendung automatisch erstellt.
- v0.6.0-alpha.1 verwendet denselben dauerhaften Produktionsschlüssel und
  aktualisiert v0.5.0-alpha.1 direkt.

### Bekannte Einschränkungen

- **Hardware-Prüfung:** Installation sowie Controller-/Netzwerktests auf einem
  echten Handheld der AYN-/Odin-/Retroid-Klasse stehen weiterhin aus.
- **Näherungsweise Dauer:** Android liefert ohne zusätzliche sensible
  Berechtigung keine präzise Nutzungsdauer fremder Apps. Der Timer misst vom
  erfolgreichen Start bis zur erneuten Sichtbarkeit des Dashboards; dadurch
  können Unterbrechungen außerhalb der Streaming-App enthalten sein.
- Es wird nur die letzte abgeschlossene Session gespeichert; dieses Release
  ergänzt noch keine vollständige Session-Historie.
- Debug-signierte Vorschauen (v0.3 und früher) können nicht direkt aktualisiert
  werden. Exportiere bei Bedarf die Konfiguration vor der Deinstallation.

### Prüfung

- 47 JVM-Tests bestanden mit 0 Fehlern und 0 übersprungenen Tests.
- 14 Android-Instrumentierungs-/Compose-Tests bestanden auf Android 9/API 28;
  dieselben 14 bestanden auf Android 15/API 35, jeweils ohne Fehler oder
  übersprungene Tests.
- Debug- und Release-Lint melden 0 Probleme; Debug-APK, optimierte Release-APK,
  Release-AAB und Android-Test-APK werden erfolgreich gebaut.

- Die Prüfung auf einem echten Handheld steht wie oben dokumentiert noch aus.

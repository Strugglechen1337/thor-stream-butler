Privacy policy

Thor Stream Butler is designed to work without an account, advertising, analytics, tracking, telemetry, or a cloud backend. This policy explains the local data the app uses and the few network requests initiated by its diagnostics.

Last updated: July 17, 2026

1. Scope and contact

This policy covers the Thor Stream Butler Android app and its project website. Questions can be submitted through the repository's public issue tracker. Please do not include private network identifiers in a public issue. Security-sensitive reports should follow the security policy.

2. Data stored on the device

Depending on the features you use, the app stores the following data only in its private Android app storage:

  • dashboard entries, Android package names, custom labels, sort order, and streaming profiles;
  • local host names, hostnames or IP addresses, optional MAC addresses and ports, Wake-on-LAN settings, and test status;
  • up to 100 recent network measurements, which may include connection type, local IP address, gateway, Wi-Fi SSID, radio information, latency, jitter, packet loss, download rate, and a tested host;
  • app settings and, only when explicitly enabled, up to 200 diagnostic event names and timestamps. Diagnostic logs contain no IP addresses, SSIDs, MAC addresses, hostnames, or credentials.

The app does not store passwords or streaming-service credentials.

3. Network processing

Thor Stream Butler does not automatically upload its database or settings. Network traffic occurs only for functionality you start or explicitly enable:

  • ping, DNS, and reachability checks contact the configured test target, the device's DNS resolver, or a host you configured;
  • local discovery uses Android Network Service Discovery on the current local network;
  • TCP checks and Wake-on-LAN packets are sent to the local target you selected;
  • the optional download test requests test bytes from speed.cloudflare.com. It is disabled by default. Cloudflare and the network path necessarily receive the request's source IP address and normal connection metadata, but Thor Stream Butler sends no saved app configuration with that request;
  • opening source-code or privacy links leaves the app and uses the browser selected on the device.

The app contains no advertising, analytics, crash-reporting, tracking, or telemetry SDK.

4. Permissions

The app declares internet, network-state, and Wi-Fi-state permissions for diagnostics. Android may additionally require Nearby Wi-Fi or location permission to expose SSID details, and Android 17 or newer requires local-network permission before host discovery, TCP tests, or Wake-on-LAN. Runtime permissions are requested when the related feature is used. Denial reduces the available values but does not create an account or transmit stored data.

5. Export, retention, and deletion

Configuration export is initiated by the user through Android's document picker. Export files can include private host and network identifiers; they are not encrypted by the app and should be stored securely. History is excluded unless the user includes it. Individual dashboard entries and hosts can be deleted in the app. Measurement history and diagnostic logs can be cleared in Settings. Uninstalling the app removes its private local data under Android's app-storage rules. Android cloud backup is disabled for this app.

6. Sharing, sale, and profiling

Thor Stream Butler does not sell data, build advertising profiles, or share its local database with the project maintainers. There is no account system and no project-operated backend receiving app data.

7. Changes

Material changes will be documented in the bilingual changelog and release notes. The current policy remains available at this URL.

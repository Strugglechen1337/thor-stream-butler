# Project instructions / Projektanweisungen

## English

- All public GitHub content must be maintained in both English and German.
- This includes the README, GitHub Pages, issue and pull-request templates,
  contribution and security documentation, changelogs, release notes, and
  release-facing files.
- English is the default GitHub Pages URL. The complete German version lives at
  `/de/`; both pages must link to each other with visible language controls and
  `hreflang` metadata.
- Do not publish or mark a release complete until English and German sections
  describe the same features, fixes, limitations, migration steps, and known issues.
- Update `CHANGELOG.md` and use `.github/RELEASE_TEMPLATE.md` for every release.
- Preserve privacy: remove IP addresses, SSIDs, MAC addresses, hostnames,
  credentials, and other personal network data from public text and images.

## Deutsch

- Alle öffentlichen GitHub-Inhalte müssen auf Englisch und Deutsch gepflegt werden.
- Das gilt für README, GitHub Pages, Issue- und Pull-Request-Vorlagen,
  Contribution- und Security-Dokumentation, Changelogs, Release Notes und alle
  releasebezogenen Dateien.
- Englisch ist die Standard-URL der GitHub Page. Die vollständige deutsche
  Variante liegt unter `/de/`; beide Seiten müssen mit sichtbaren
  Sprachumschaltern und `hreflang`-Metadaten aufeinander verweisen.
- Ein Release darf erst veröffentlicht oder als vollständig markiert werden,
  wenn Englisch und Deutsch dieselben Funktionen, Fehlerbehebungen,
  Einschränkungen, Migrationsschritte und bekannten Probleme beschreiben.
- Für jedes Release `CHANGELOG.md` aktualisieren und
  `.github/RELEASE_TEMPLATE.md` verwenden.
- Datenschutz wahren: IP-Adressen, SSIDs, MAC-Adressen, Hostnamen, Zugangsdaten
  und andere persönliche Netzwerkdaten aus öffentlichen Texten und Bildern entfernen.
